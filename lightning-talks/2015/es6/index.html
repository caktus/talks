JavaScript ES6 "The Good Parts"

  • Like Javascript, there is a lot of ES6 but its good to focus.
  • Not really "bad parts" like its ancestor, but we'll focus on the most important changes here.
  • For parts I don't cover, there is a complete feature list available in the talk notes as follow up!

What's up with the name?

  • LiveScript ← What Netscape made
  • JavaScript ← What they and Sun renamed it to
  • ECMAScript ← "Official" name from standards body
  • ECMAScript 5 ← What we use today
  • ECMAScript 6 (ES6) ← What this was called
  • ECMAScript 2015 ← What we've bikeshedded ourselves to
Note: Current Javascript is "ECMAScript 5" we've already been changing, ES6 is just a bigger jump. I'll say "ES6" because I don't have time to pronounce anything longer.

Extend from any expression!


function mixin(base, ...mixins) {
    class CombinedBase extends base
    while (mixins.length > 0) {
        class CombinedBase extends mixins.splice(0, 1)[0]
    }
    return CombinedBase
}
class PrivateView extends mixin(View, RequireLogin, RequireSSH) {
    get template() {
        return "login.html"
    }
}
No instance properties in classes, just methods or getters/setters.
