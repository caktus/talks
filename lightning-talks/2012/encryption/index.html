Traditional encryption

  • One key
  • Both parties need it
  • If anybody else finds it out, they can read the messages

Enigma machine

Key distribution

  • Both parties need the same key
  • It has to be kept secret from anyone else
  • Before they can communicate securely...
  • They first need to communicate securely

RSA To the Rescue

  • 1977
  • Ron Rivest, Adi Shamir and Leonard Adleman

Two keys!

The keys are interchangeable

  • You can encrypt with either key
  • and then decrypt with the other

Naive usage

  • Alice generates key pair, gives Key 2 to Bob
  • encrypts with Key 1
  • ... but that doesn't help at all!
  • Still have to get Key 2 to Bob securely

Smart usage - turn it around

Alice generates key pair, gives Key 2 to Bob.

Bob encrypts messages with Key 2 and sends to Alice

Alice decrypts with Key 1

Digital signature

We still have a problem

Whose key?

Trust for hire

  • Entities the community trusts
  • will verify:
    • the person with this key
    • is Alice
  • and then vouch:
    • this key
    • is Alice's
  • We call them Certificate Authorities

What's a certificate?

  • Verisign doesn't want people calling them:
  • "What's Alice's key?"
  • So they give Alice a certificate
  • to prove that Verisign vouches for Alice's key

Certificates

A certificate has three things:

  1. An identity:
    • "www.amazon.com" or
    • "obama@whitehouse.gov"
  2. A public key
  3. A certificate authority's digital signature

SSL

  • When our browsers establish an SSL connection
  • the server sends them a certificate
  • and they verify it was signed
  • by a certificate authority they trust

SSL part 2

  • Browsers come with a list of:
    • certificate authorities they trust
    • and their public keys
  • So that Bob can buy his kids presents
    online with confidence

All Hail RSA!

  • Forget Caesar
  • Without RSA, a good part of our economy
  • would not exist
